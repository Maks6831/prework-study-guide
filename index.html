  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.

CSS

  • There are three ways to style a webpage using CSS: inline CSS, internal CSS style sheet, and external CSS style sheet.
  • Inline styles apply to a specific HTML tag, using a style attribute with a CSS rule to style a particular page element only.
  • Internal styles make it possible to embed an entire stylesheet directly in the HTML file.
  • Using an external CSS style sheet is the most common way to style a website as it allows one to keep their CSS rules in a seperate file.
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • The * selector is used to the margin and padding for the all the elements visible on a page.
  • You can combine two selectors into a single CSS using a comma.
  • A period preceding a selector designates this to be a class selector, any HTML element assigned to the class will gain those property assignments.

Git

  • git clone copied-SSH-value: clones GitHub repository onto your computer.
  • git status: checks what branch we are currently on.
  • git checkout -b branch-name: creates a new branch and switches to it.
  • git add -A: add command adds modifications to the staging area, -A indicates that we want to add all changes.
  • The staging area informs Git that the changes will be added in the next commit.
  • git commit -m "add starter code": commit comand with a commit message that describes what the commit contains.
  • git pull origin main: pulls the source code from GitHub to ensure we have latest commits and changes.(good practice).
  • git push origin branch-name: pushes your changes to GitHub. Make sure to not push to main branch.
  • git checkout main: switches to main branch.
